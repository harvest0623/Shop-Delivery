### 1.1 系统概述
外卖商城系统是一个基于Web的在线订餐平台，为用户提供便捷的订餐服务，为商家提供商品管理和销售统计功能。

### 1.3 数据库需求分析

#### 实体识别
1. **商家（Shop）**：提供外卖服务的餐饮商家
2. **商品分类（Category）**：商品的分类信息
3. **商品（Product）**：商家提供的具体商品
4. **顾客（Customer）**：使用系统的注册用户
5. **地址（Address）**：顾客的收货地址
6. **购物车（Cart）**：顾客选购的商品临时存储
7. **订单（Order）**：顾客提交的购买订单
8. **订单项（OrderItem）**：订单中的具体商品

#### 实体属性

| 实体 | 属性 |
|------|------|
| 商家 | id, name, description, address, phone, image_url, rating, delivery_time, delivery_fee, min_order, status |
| 商品分类 | id, name, icon, sort_order |
| 商品 | id, shop_id, category_id, name, description, price, image_url, stock, is_featured |
| 顾客 | id, username, password, email, phone, nickname, avatar_url |
| 地址 | id, customer_id, name, phone, province, city, district, detail, is_default |
| 购物车 | id, customer_id, product_id, quantity |
| 订单 | id, customer_id, shop_id, address_id, total_amount, delivery_fee, status, order_no, remark, return_reason |
| 订单项 | id, order_id, product_id, quantity, price |

#### 实体间联系
- 商家与商品：一对多（一个商家有多个商品）
- 商品分类与商品：一对多（一个分类包含多个商品）
- 顾客与地址：一对多（一个顾客有多个地址）
- 顾客与购物车：一对多（一个顾客有多个购物车项）
- 顾客与订单：一对多（一个顾客有多个订单）
- 商家与订单：一对多（一个商家有多个订单）
- 订单与订单项：一对多（一个订单包含多个订单项）
- 商品与订单项：一对多（一个商品可出现在多个订单项中）

## 三、逻辑结构设计（关系模式）

### 3.1 关系模式定义

```sql
-- 商家表
shops(id, name, description, address, phone, image_url, rating, 
      delivery_time, delivery_fee, min_order, status)
主键：id

-- 商品分类表
categories(id, name, icon, sort_order)
主键：id

-- 商品表
products(id, shop_id, category_id, name, description, price, 
         image_url, stock, is_featured)
主键：id
外键：shop_id → shops(id)
外键：category_id → categories(id)

-- 顾客表
customers(id, username, password, email, phone, nickname, avatar_url)
主键：id

-- 地址表
addresses(id, customer_id, name, phone, province, city, district, 
          detail, is_default)
主键：id
外键：customer_id → customers(id)

-- 购物车表
cart(id, customer_id, product_id, quantity)
主键：id
外键：customer_id → customers(id)
外键：product_id → products(id)

-- 订单表
orders(id, customer_id, shop_id, address_id, total_amount, 
       delivery_fee, status, order_no, remark, return_reason)
主键：id
外键：customer_id → customers(id)
外键：shop_id → shops(id)
外键：address_id → addresses(id)

-- 订单项表
order_items(id, order_id, product_id, quantity, price)
主键：id
外键：order_id → orders(id)
外键：product_id → products(id)
```

### 3.2 关系模式规范化

所有关系模式均满足第三范式（3NF）：
- 每个属性都是原子的（1NF）
- 不存在非主属性对主键的部分依赖（2NF）
- 不存在非主属性对主键的传递依赖（3NF）

## 六、视图创建

### 6.1 订单详情视图（order_detail_view）

**功能描述**：整合订单、顾客、商家、地址信息，方便查看订单完整详情。

```sql
CREATE VIEW order_detail_view AS
SELECT 
    o.id AS order_id,
    o.order_no,
    o.total_amount,
    o.delivery_fee,
    o.status,
    o.remark,
    o.return_reason,
    o.created_at,
    c.username,
    c.phone AS customer_phone,
    c.nickname,
    s.name AS shop_name,
    s.image_url AS shop_image,
    CONCAT(a.province, a.city, a.district, a.detail) AS address_detail,
    a.name AS receiver_name,
    a.phone AS receiver_phone
FROM orders o
JOIN customers c ON o.customer_id = c.id
JOIN shops s ON o.shop_id = s.id
LEFT JOIN addresses a ON o.address_id = a.id;
```

### 6.2 商家商品视图（shop_product_view）

### 6.3 销售汇总视图（sales_summary_view）

**功能描述**：按商家汇总销售数据，包括订单数、销售额、退货额、净销售额等。

```sql
CREATE VIEW sales_summary_view AS
SELECT 
    s.id AS shop_id,
    s.name AS shop_name,
    COUNT(DISTINCT o.id) AS total_orders,
    SUM(CASE WHEN o.status = 'completed' THEN o.total_amount ELSE 0 END) AS total_sales,
    SUM(CASE WHEN o.status = 'returned' THEN o.total_amount ELSE 0 END) AS total_returns,
    SUM(CASE WHEN o.status = 'completed' THEN o.total_amount ELSE 0 END) - 
    SUM(CASE WHEN o.status = 'returned' THEN o.total_amount ELSE 0 END) AS net_sales,
    COUNT(CASE WHEN o.status = 'completed' THEN 1 END) AS completed_orders,
    COUNT(CASE WHEN o.status = 'returned' THEN 1 END) AS returned_orders,
    COUNT(CASE WHEN o.status = 'cancelled' THEN 1 END) AS cancelled_orders
FROM shops s
LEFT JOIN orders o ON s.id = o.shop_id
GROUP BY s.id, s.name;
```

### 6.4 顾客订单视图（customer_order_view）

**功能描述**：按顾客汇总订单数据，包括订单数、消费总额、最后下单时间。

```sql
CREATE VIEW customer_order_view AS
SELECT 
    c.id AS customer_id,
    c.username,
    c.nickname,
    COUNT(o.id) AS total_orders,
    SUM(CASE WHEN o.status = 'completed' THEN o.total_amount ELSE 0 END) AS total_spent,
    MAX(o.created_at) AS last_order_date
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id
GROUP BY c.id, c.username, c.nickname;
```

## 七、存储过程

### 7.1 创建订单存储过程（CreateOrder）

**功能描述**：创建新订单，自动生成订单编号，返回订单ID和订单号。

```sql
DELIMITER //
CREATE PROCEDURE CreateOrder(
    IN p_customer_id INT,
    IN p_shop_id INT,
    IN p_address_id INT,
    IN p_total_amount DECIMAL(10,2),
    IN p_delivery_fee DECIMAL(5,2),
    IN p_remark TEXT
)
BEGIN
    DECLARE v_order_no VARCHAR(50);
    DECLARE v_order_id INT;
    
    SET v_order_no = CONCAT('ORD', DATE_FORMAT(NOW(), '%Y%m%d'), 
                            LPAD(FLOOR(RAND() * 10000), 4, '0'));
    
    INSERT INTO orders (customer_id, shop_id, address_id, total_amount, 
                        delivery_fee, status, order_no, remark)
    VALUES (p_customer_id, p_shop_id, p_address_id, p_total_amount, 
            p_delivery_fee, 'pending', v_order_no, p_remark);
    
    SET v_order_id = LAST_INSERT_ID();
    
    SELECT v_order_id AS order_id, v_order_no AS order_no;
END //
DELIMITER ;
```

### 7.2 处理退货存储过程（ProcessReturn）

**功能描述**：处理订单退货申请，验证订单状态是否允许退货，更新订单状态为退货。

```sql
DELIMITER //
CREATE PROCEDURE ProcessReturn(
    IN p_order_id INT,
    IN p_reason VARCHAR(255)
)
BEGIN
    DECLARE v_status VARCHAR(20);
    
    SELECT status INTO v_status FROM orders WHERE id = p_order_id;
    
    IF v_status = 'completed' THEN
        UPDATE orders 
        SET status = 'returned', return_reason = p_reason, updated_at = NOW()
        WHERE id = p_order_id;
        
        SELECT 'Return Success' AS message, p_order_id AS order_id;
    ELSE
        SELECT 'Order status does not allow return' AS message, p_order_id AS order_id;
    END IF;
END //
DELIMITER ;
```

### 7.3 获取销售统计存储过程（GetSalesStatistics）

**功能描述**：根据日期范围查询订单统计数据，包括订单数、金额、各状态订单数。

```sql
DELIMITER //
CREATE PROCEDURE GetSalesStatistics(
    IN p_start_date DATE,
    IN p_end_date DATE
)
BEGIN
    SELECT 
        COUNT(*) AS total_orders,
        SUM(total_amount) AS total_amount,
        SUM(delivery_fee) AS total_delivery_fee,
        COUNT(CASE WHEN status = 'completed' THEN 1 END) AS completed_count,
        COUNT(CASE WHEN status = 'returned' THEN 1 END) AS returned_count,
        COUNT(CASE WHEN status = 'cancelled' THEN 1 END) AS cancelled_count,
        COUNT(CASE WHEN status = 'pending' THEN 1 END) AS pending_count
    FROM orders
    WHERE DATE(created_at) BETWEEN p_start_date AND p_end_date;
END //
DELIMITER ;
```

### 7.4 添加购物车存储过程（AddToCart）

**功能描述**：添加商品到购物车，检查库存是否充足，如果商品已在购物车则增加数量。

```sql
DELIMITER //
CREATE PROCEDURE AddToCart(
    IN p_customer_id INT,
    IN p_product_id INT,
    IN p_quantity INT
)
BEGIN
    DECLARE v_stock INT;
    DECLARE v_exists INT;
    
    SELECT stock INTO v_stock FROM products WHERE id = p_product_id;
    
    IF v_stock < p_quantity THEN
        SELECT 'Insufficient stock' AS message;
    ELSE
        SELECT COUNT(*) INTO v_exists FROM cart 
        WHERE customer_id = p_customer_id AND product_id = p_product_id;
        
        IF v_exists > 0 THEN
            UPDATE cart SET quantity = quantity + p_quantity, updated_at = NOW()
            WHERE customer_id = p_customer_id AND product_id = p_product_id;
        ELSE
            INSERT INTO cart (customer_id, product_id, quantity)
            VALUES (p_customer_id, p_product_id, p_quantity);
        END IF;
        
        SELECT 'Added successfully' AS message;
    END IF;
END //
DELIMITER ;
```

## 八、触发器

### 8.1 订单创建时减少库存触发器

**功能描述**：当订单项被插入时，自动减少对应商品的库存数量。

```sql
DELIMITER //
CREATE TRIGGER trg_after_order_insert
AFTER INSERT ON order_items
FOR EACH ROW
BEGIN
    UPDATE products 
    SET stock = stock - NEW.quantity
    WHERE id = NEW.product_id;
END //
DELIMITER ;
```

### 8.2 订单退货时恢复库存触发器

**功能描述**：当订单状态更新为退货时，自动恢复对应商品的库存数量。

```sql
DELIMITER //
CREATE TRIGGER trg_after_order_return
AFTER UPDATE ON orders
FOR EACH ROW
BEGIN
    IF OLD.status != 'returned' AND NEW.status = 'returned' THEN
        UPDATE products p
        JOIN order_items oi ON p.id = oi.product_id
        SET p.stock = p.stock + oi.quantity
        WHERE oi.order_id = NEW.id;
    END IF;
END //
DELIMITER ;
```

### 8.3 订单取消时恢复库存触发器

**功能描述**：当订单状态更新为取消时，自动恢复对应商品的库存数量。

```sql
DELIMITER //
CREATE TRIGGER trg_after_order_cancel
AFTER UPDATE ON orders
FOR EACH ROW
BEGIN
    IF OLD.status != 'cancelled' AND NEW.status = 'cancelled' THEN
        UPDATE products p
        JOIN order_items oi ON p.id = oi.product_id
        SET p.stock = p.stock + oi.quantity
        WHERE oi.order_id = NEW.id;
    END IF;
END //
DELIMITER ;
```
