## 一、系统功能描述、功能模块图、数据库需求分析

### 1.1 系统功能描述

外卖商城系统是一个基于Web的在线订餐平台，为用户提供便捷的订餐服务。系统包含以下核心功能：

1. **用户管理**：用户注册、登录、个人信息管理、收货地址管理
2. **商品浏览**：按分类浏览商品、搜索商品、查看商品详情
3. **商家浏览**：查看商家列表、商家详情、商家商品
4. **购物车管理**：添加商品、修改数量、删除商品、清空购物车
5. **订单管理**：提交订单、查看订单、取消订单、申请退货
6. **销售统计**：订单统计、热销商品排行、热门商家排行、销售趋势分析

### 1.3 数据库需求分析

#### 1.3.1 实体识别

通过分析系统功能，识别出以下8个实体：

| 序号 | 实体名称 | 实体说明 |
|------|---------|---------|
| 1 | 商家(Shop) | 提供外卖服务的餐饮商家 |
| 2 | 商品分类(Category) | 商品的分类信息 |
| 3 | 商品(Product) | 商家提供的具体商品 |
| 4 | 顾客(Customer) | 使用系统的注册用户 |
| 5 | 地址(Address) | 顾客的收货地址 |
| 6 | 购物车(Cart) | 顾客选购的商品临时存储 |
| 7 | 订单(Order) | 顾客提交的购买订单 |
| 8 | 订单项(OrderItem) | 订单中的具体商品 |

#### 1.3.2 实体属性

**商家(Shop)实体属性：**
- 商家ID(id)：唯一标识每个商家
- 商家名称(name)：商家的名称
- 商家描述(description)：商家的详细介绍
- 商家地址(address)：商家的实体地址
- 联系电话(phone)：商家的联系电话
- 商家图片(image_url)：商家的展示图片
- 评分(rating)：商家的用户评分（1-5分）
- 配送时间(delivery_time)：平均配送时间（分钟）
- 配送费(delivery_fee)：配送费用
- 起送价(min_order)：最低起送金额
- 状态(status)：营业状态（营业/休息）
- 创建时间(created_at)：商家入驻时间

**商品分类(Category)实体属性：**
- 分类ID(id)：唯一标识每个分类
- 分类名称(name)：分类的名称（如：汉堡、饮品）
- 图标(icon)：分类的图标
- 排序(sort_order)：分类的显示顺序
- 创建时间(created_at)：分类创建时间

**商品(Product)实体属性：**
- 商品ID(id)：唯一标识每个商品
- 商家ID(shop_id)：所属商家
- 分类ID(category_id)：所属分类
- 商品名称(name)：商品的名称
- 商品描述(description)：商品的详细介绍
- 价格(price)：商品的售价
- 商品图片(image_url)：商品的展示图片
- 库存(stock)：商品的库存数量
- 是否招牌(is_featured)：是否为招牌商品
- 创建时间(created_at)：商品上架时间

**顾客(Customer)实体属性：**
- 顾客ID(id)：唯一标识每个顾客
- 用户名(username)：登录用户名
- 密码(password)：登录密码（加密存储）
- 邮箱(email)：联系邮箱
- 手机号(phone)：联系手机号
- 昵称(nickname)：显示昵称
- 头像(avatar_url)：用户头像
- 创建时间(created_at)：注册时间

**地址(Address)实体属性：**
- 地址ID(id)：唯一标识每个地址
- 顾客ID(customer_id)：所属顾客
- 收件人姓名(name)：收件人姓名
- 收件人电话(phone)：收件人电话
- 省份(province)：所在省份
- 城市(city)：所在城市
- 区县(district)：所在区县
- 详细地址(detail)：街道门牌号等详细信息
- 是否默认(is_default)：是否为默认地址
- 创建时间(created_at)：地址添加时间

**购物车(Cart)实体属性：**
- 购物车ID(id)：唯一标识每条记录
- 顾客ID(customer_id)：所属顾客
- 商品ID(product_id)：所选商品
- 数量(quantity)：购买数量
- 创建时间(created_at)：添加时间
- 更新时间(updated_at)：最后修改时间

**订单(Order)实体属性：**
- 订单ID(id)：唯一标识每个订单
- 顾客ID(customer_id)：下单顾客
- 商家ID(shop_id)：订单所属商家
- 地址ID(address_id)：配送地址
- 订单金额(total_amount)：商品总金额
- 配送费(delivery_fee)：配送费用
- 订单状态(status)：订单当前状态
- 订单编号(order_no)：唯一的订单编号
- 备注(remark)：订单备注
- 退货原因(return_reason)：退货原因
- 创建时间(created_at)：下单时间
- 更新时间(updated_at)：最后更新时间

**订单项(OrderItem)实体属性：**
- 订单项ID(id)：唯一标识每条记录
- 订单ID(order_id)：所属订单
- 商品ID(product_id)：所购商品
- 数量(quantity)：购买数量
- 单价(price)：下单时的商品单价

#### 1.3.3 实体之间的联系

| 联系 | 实体1 | 实体2 | 联系类型 | 说明 |
|------|-------|-------|---------|------|
| 属于 | 商品 | 商家 | 多对一 | 一个商家有多个商品 |
| 属于 | 商品 | 商品分类 | 多对一 | 一个分类包含多个商品 |
| 拥有 | 顾客 | 地址 | 一对多 | 一个顾客有多个地址 |
| 拥有 | 顾客 | 购物车 | 一对多 | 一个顾客有多个购物车项 |
| 拥有 | 顾客 | 订单 | 一对多 | 一个顾客有多个订单 |
| 拥有 | 商家 | 订单 | 一对多 | 一个商家有多个订单 |
| 包含 | 订单 | 订单项 | 一对多 | 一个订单包含多个订单项 |
| 属于 | 订单项 | 商品 | 多对一 | 一个商品可出现在多个订单项中 |

## 三、逻辑结构设计：关系模式

### 3.1 E-R图转换为关系模式

将E-R图中的每个实体和联系转换为关系模式：

#### 1) 商家表（shops）
```sql
shops(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    name VARCHAR(100) NOT NULL,
    description TEXT,
    address VARCHAR(255),
    phone VARCHAR(20),
    image_url VARCHAR(500),
    rating DECIMAL(2,1) DEFAULT 4.5,
    delivery_time INT DEFAULT 30,
    delivery_fee DECIMAL(5,2) DEFAULT 3.00,
    min_order DECIMAL(5,2) DEFAULT 20.00,
    status VARCHAR(20) DEFAULT 'open',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

#### 2) 商品分类表（categories）
```sql
categories(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    name VARCHAR(50) NOT NULL,
    icon VARCHAR(100),
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

#### 3) 商品表（products）
```sql
products(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    shop_id INT NOT NULL,               -- 外键→shops(id)
    category_id INT,                    -- 外键→categories(id)
    name VARCHAR(100) NOT NULL,
    description TEXT,
    price DECIMAL(10,2) NOT NULL,
    image_url VARCHAR(500),
    stock INT DEFAULT 100,
    is_featured TINYINT(1) DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (shop_id) REFERENCES shops(id),
    FOREIGN KEY (category_id) REFERENCES categories(id)
)
```

#### 4) 顾客表（customers）
```sql
customers(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    phone VARCHAR(20),
    nickname VARCHAR(50),
    avatar_url VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

#### 5) 地址表（addresses）
```sql
addresses(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    customer_id INT NOT NULL,           -- 外键→customers(id)
    name VARCHAR(50),
    phone VARCHAR(20),
    province VARCHAR(50),
    city VARCHAR(50),
    district VARCHAR(50),
    detail VARCHAR(255),
    is_default TINYINT(1) DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
)
```

#### 6) 购物车表（cart）
```sql
cart(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    customer_id INT NOT NULL,           -- 外键→customers(id)
    product_id INT NOT NULL,            -- 外键→products(id)
    quantity INT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id),
    FOREIGN KEY (product_id) REFERENCES products(id)
)
```

#### 7) 订单表（orders）
```sql
orders(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    customer_id INT NOT NULL,           -- 外键→customers(id)
    shop_id INT NOT NULL,               -- 外键→shops(id)
    address_id INT,                     -- 外键→addresses(id)
    total_amount DECIMAL(10,2) NOT NULL,
    delivery_fee DECIMAL(5,2) DEFAULT 0.00,
    status VARCHAR(20) DEFAULT 'pending',
    order_no VARCHAR(50) UNIQUE,
    remark TEXT,
    return_reason VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id),
    FOREIGN KEY (shop_id) REFERENCES shops(id),
    FOREIGN KEY (address_id) REFERENCES addresses(id)
)
```

#### 8) 订单项表（order_items）
```sql
order_items(
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 主键
    order_id INT NOT NULL,              -- 外键→orders(id)
    product_id INT NOT NULL,            -- 外键→products(id)
    quantity INT DEFAULT 1,
    price DECIMAL(10,2) NOT NULL,
    FOREIGN KEY (order_id) REFERENCES orders(id),
    FOREIGN KEY (product_id) REFERENCES products(id)
)
```

### 3.2 主键、外键汇总表

| 表名 | 主键 | 外键 | 外键引用表 |
|------|------|------|-----------|
| shops | id | 无 | - |
| categories | id | 无 | - |
| products | id | shop_id, category_id | shops(id), categories(id) |
| customers | id | 无 | - |
| addresses | id | customer_id | customers(id) |
| cart | id | customer_id, product_id | customers(id), products(id) |
| orders | id | customer_id, shop_id, address_id | customers(id), shops(id), addresses(id) |
| order_items | id | order_id, product_id | orders(id), products(id) |

### 3.3 关系模式规范化说明

所有关系模式均满足第三范式（3NF）：

**第一范式（1NF）**：每个属性都是原子的，不可再分。所有表的每个字段都是单一值。

**第二范式（2NF）**：满足1NF，且不存在非主属性对主键的部分依赖。所有表都有单字段主键（id），不存在部分依赖。

**第三范式（3NF）**：满足2NF，且不存在非主属性对主键的传递依赖。
- 例如：订单表中的 `total_amount` 虽然可以通过订单项计算得出，但为了查询效率和数据一致性，将其作为冗余字段存储，并通过业务逻辑保证一致性。

## 四、创建用户、授予权限

### 4.1 创建数据库用户

```sql
-- 创建用户，设置密码
CREATE USER 'shop_admin'@'localhost' IDENTIFIED BY 'shop123456';

-- 授予数据库所有权限
GRANT ALL PRIVILEGES ON shop_delivery.* TO 'shop_admin'@'localhost';

-- 刷新权限使其生效
FLUSH PRIVILEGES;
```

### 4.2 权限说明

| 权限类型 | 说明 |
|---------|------|
| ALL PRIVILEGES | 授予用户对数据库的所有操作权限，包括SELECT、INSERT、UPDATE、DELETE、CREATE、DROP等 |
| ON shop_delivery.* | 权限作用于shop_delivery数据库的所有表 |
| TO 'shop_admin'@'localhost' | 只允许从本机登录的shop_admin用户 |

### 4.3 验证用户创建

```sql
-- 查看用户权限
SHOW GRANTS FOR 'shop_admin'@'localhost';

-- 使用新用户登录
mysql -u shop_admin -p

-- 登录后创建数据库
CREATE DATABASE shop_delivery;
```

## 七、核心业务功能用存储过程实现

### 7.1 存储过程1：CreateOrder（创建订单）

**功能描述**：创建新订单，自动生成唯一的订单编号，返回订单ID和订单号。

**实现代码**：
```sql
DELIMITER //
CREATE PROCEDURE CreateOrder(
    IN p_customer_id INT,           -- 顾客ID
    IN p_shop_id INT,               -- 商家ID
    IN p_address_id INT,            -- 地址ID
    IN p_total_amount DECIMAL(10,2), -- 订单金额
    IN p_delivery_fee DECIMAL(5,2),  -- 配送费
    IN p_remark TEXT                 -- 备注
)
BEGIN
    DECLARE v_order_no VARCHAR(50);
    DECLARE v_order_id INT;
    
    -- 自动生成订单编号：ORD + 年月日 + 4位随机数
    -- 例如：ORD202605193295
    SET v_order_no = CONCAT('ORD', DATE_FORMAT(NOW(), '%Y%m%d'), 
                            LPAD(FLOOR(RAND() * 10000), 4, '0'));
    
    -- 插入订单记录
    INSERT INTO orders (customer_id, shop_id, address_id, total_amount, 
                        delivery_fee, status, order_no, remark)
    VALUES (p_customer_id, p_shop_id, p_address_id, p_total_amount, 
            p_delivery_fee, 'pending', v_order_no, p_remark);
    
    -- 获取新插入的订单ID
    SET v_order_id = LAST_INSERT_ID();
    
    -- 返回订单ID和订单号
    SELECT v_order_id AS order_id, v_order_no AS order_no;
END //
DELIMITER ;
```

**调用示例**：
```sql
CALL CreateOrder(1, 1, 1, 60.00, 3.00, '请尽快送达');
```

### 7.2 存储过程2：ProcessReturn（处理退货）

**功能描述**：处理订单退货申请，验证订单状态是否为"已完成"，如果是则更新为"已退货"并记录退货原因。

**实现代码**：
```sql
DELIMITER //
CREATE PROCEDURE ProcessReturn(
    IN p_order_id INT,           -- 订单ID
    IN p_reason VARCHAR(255)     -- 退货原因
)
BEGIN
    DECLARE v_status VARCHAR(20);
    DECLARE v_message VARCHAR(100);
    
    -- 获取订单当前状态
    SELECT status INTO v_status FROM orders WHERE id = p_order_id;
    
    -- 判断订单状态是否允许退货（只有已完成的订单才能退货）
    IF v_status = 'completed' THEN
        UPDATE orders 
        SET status = 'returned', 
            return_reason = p_reason,
            updated_at = NOW()
        WHERE id = p_order_id;
        
        SET v_message = '退货申请成功';
    ELSE
        SET v_message = '订单状态不允许退货';
    END IF;
    
    -- 返回处理结果
    SELECT v_message AS message, p_order_id AS order_id, v_status AS original_status;
END //
DELIMITER ;
```

**调用示例**：
```sql
CALL ProcessReturn(1, '商品质量问题，需要退货');
```

### 7.3 存储过程3：GetSalesStatistics（获取销售统计）

**功能描述**：根据指定的日期范围查询订单统计数据，包括订单总数、总金额、各状态订单数量。

**实现代码**：
```sql
DELIMITER //
CREATE PROCEDURE GetSalesStatistics(
    IN p_start_date DATE,    -- 开始日期
    IN p_end_date DATE       -- 结束日期
)
BEGIN
    SELECT 
        COUNT(*) AS total_orders,
        SUM(total_amount) AS total_amount,
        SUM(delivery_fee) AS total_delivery_fee,
        COUNT(CASE WHEN status = 'completed' THEN 1 END) AS completed_count,
        COUNT(CASE WHEN status = 'returned' THEN 1 END) AS returned_count,
        COUNT(CASE WHEN status = 'cancelled' THEN 1 END) AS cancelled_count,
        COUNT(CASE WHEN status = 'pending' THEN 1 END) AS pending_count,
        SUM(CASE WHEN status = 'completed' THEN total_amount ELSE 0 END) AS completed_amount,
        SUM(CASE WHEN status = 'returned' THEN total_amount ELSE 0 END) AS returned_amount
    FROM orders
    WHERE DATE(created_at) BETWEEN p_start_date AND p_end_date;
END //
DELIMITER ;
```

**调用示例**：
```sql
-- 查询2024年1月的销售统计
CALL GetSalesStatistics('2024-01-01', '2024-01-31');
```

### 7.4 存储过程4：AddToCart（添加购物车）

**功能描述**：添加商品到购物车，先检查库存是否充足，如果商品已在购物车中则增加数量，否则新增记录。

**实现代码**：
```sql
DELIMITER //
CREATE PROCEDURE AddToCart(
    IN p_customer_id INT,    -- 顾客ID
    IN p_product_id INT,     -- 商品ID
    IN p_quantity INT        -- 数量
)
BEGIN
    DECLARE v_stock INT;
    DECLARE v_exists INT;
    DECLARE v_message VARCHAR(100);
    
    -- 检查商品库存
    SELECT stock INTO v_stock FROM products WHERE id = p_product_id;
    
    -- 库存不足
    IF v_stock < p_quantity THEN
        SET v_message = '库存不足';
    ELSE
        -- 检查购物车中是否已有该商品
        SELECT COUNT(*) INTO v_exists FROM cart 
        WHERE customer_id = p_customer_id AND product_id = p_product_id;
        
        IF v_exists > 0 THEN
            -- 已存在，增加数量
            UPDATE cart 
            SET quantity = quantity + p_quantity, 
                updated_at = NOW()
            WHERE customer_id = p_customer_id AND product_id = p_product_id;
            SET v_message = '数量已更新';
        ELSE
            -- 不存在，新增记录
            INSERT INTO cart (customer_id, product_id, quantity)
            VALUES (p_customer_id, p_product_id, p_quantity);
            SET v_message = '添加成功';
        END IF;
    END IF;
    
    -- 返回结果
    SELECT v_message AS message;
END //
DELIMITER ;
```

**调用示例**：
```sql
CALL AddToCart(1, 1, 2);
```

## 八、创建触发器保证数据正确性

### 8.1 触发器1：trg_after_order_insert（订单创建减库存）

**功能描述**：当订单项被插入时（即用户下单后），自动减少对应商品的库存数量。保证库存数据的实时性和准确性。

**实现代码**：
```sql
DELIMITER //
CREATE TRIGGER trg_after_order_insert
AFTER INSERT ON order_items
FOR EACH ROW
BEGIN
    -- 减少商品库存
    UPDATE products 
    SET stock = stock - NEW.quantity
    WHERE id = NEW.product_id;
END //
DELIMITER ;
```

**触发时机**：AFTER INSERT（在订单项插入之后触发）
**触发事件**：INSERT INTO order_items
**作用**：自动维护库存一致性，避免手动更新库存导致的数据错误

### 8.2 触发器2：trg_after_order_return（订单退货恢复库存）

**功能描述**：当订单状态从其他状态更新为"returned"（已退货）时，自动恢复该订单中所有商品的库存数量。

**实现代码**：
```sql
DELIMITER //
CREATE TRIGGER trg_after_order_return
AFTER UPDATE ON orders
FOR EACH ROW
BEGIN
    -- 判断订单状态是否变为"returned"
    IF OLD.status != 'returned' AND NEW.status = 'returned' THEN
        -- 恢复该订单所有商品的库存
        UPDATE products p
        JOIN order_items oi ON p.id = oi.product_id
        SET p.stock = p.stock + oi.quantity
        WHERE oi.order_id = NEW.id;
    END IF;
END //
DELIMITER ;
```

**触发时机**：AFTER UPDATE（在订单更新之后触发）
**触发条件**：OLD.status != 'returned' AND NEW.status = 'returned'
**作用**：退货时自动恢复库存，保证库存数据与订单状态一致

### 8.3 触发器3：trg_after_order_cancel（订单取消恢复库存）

**功能描述**：当订单状态从其他状态更新为"cancelled"（已取消）时，自动恢复该订单中所有商品的库存数量。

**实现代码**：
```sql
DELIMITER //
CREATE TRIGGER trg_after_order_cancel
AFTER UPDATE ON orders
FOR EACH ROW
BEGIN
    -- 判断订单状态是否变为"cancelled"
    IF OLD.status != 'cancelled' AND NEW.status = 'cancelled' THEN
        -- 恢复该订单所有商品的库存
        UPDATE products p
        JOIN order_items oi ON p.id = oi.product_id
        SET p.stock = p.stock + oi.quantity
        WHERE oi.order_id = NEW.id;
    END IF;
END //
DELIMITER ;
```

**触发时机**：AFTER UPDATE（在订单更新之后触发）
**触发条件**：OLD.status != 'cancelled' AND NEW.status = 'cancelled'
**作用**：取消订单时自动恢复库存，避免库存被无效占用

### 8.4 触发器的作用总结

| 触发器 | 触发事件 | 功能 | 数据一致性保障 |
|--------|---------|------|--------------|
| trg_after_order_insert | 插入订单项 | 减少商品库存 | 下单后库存自动扣减 |
| trg_after_order_return | 更新订单状态为returned | 恢复商品库存 | 退货后库存自动恢复 |
| trg_after_order_cancel | 更新订单状态为cancelled | 恢复商品库存 | 取消后库存自动恢复 |

**答辩要点**：
1. 触发器是自动执行的存储过程，无需应用程序手动调用
2. 触发器保证了库存数据的自动维护，避免人为遗漏
3. 使用触发器而不是应用程序代码维护库存，可以保证无论通过什么方式修改订单（如直接SQL操作），库存都会被正确更新
4. 触发器在数据库层面保证了数据的完整性和一致性
